# 风格指南（style guide）

> 目标：把“叙事口吻、用词偏好、标点与禁忌”固化，减少长篇后期的文风漂移。

## 叙事约束

- 人称：……
- 时态：……
- 叙述距离：贴近/中景/远景（自填）

## 起点连载节奏（可按类型调整）

- 每章至少一个“爽点/推进点/信息增量”
- 章末尽量落“钩子”（但避免纯标题党）
- 关键名词首次出现要清晰、可复述

## 角色声线

- 主角口癖：……
- 关键配角口癖：……
- 对白风格：短促/迂回/强势/阴阳（自填）

## 标点与格式

- 引号：中文引号“”为主
- 省略号：……（统一）
- 数字：阿拉伯/中文（统一）

## 正文禁忌（强约束）

- 正文中不出现任何索引 ID（如 char-/loc-/fac-/item-/sys-/thr-/evt-）。
- 不写“系统面板/提示”类括号文本（如 `【提示：...】`、`【资源点线索已获得】`），除非本书明确是系统流并由你要求。

## 禁忌与敏感点

- 平台红线清单：……
- 主动规避策略：……
